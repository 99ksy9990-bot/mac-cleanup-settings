# 맥 정리/설정 프로젝트

이 프로젝트는 맥 사용 환경을 깔끔하게 유지하고, 자주 쓰는 설정을 편하게 다듬는 작업용입니다.

## Codex 사용 기준

이 문서는 과거 정리 기록과 참고 기준입니다.
새 작업에서는 이 기준을 반드시 따르지 말고, 사용자의 현재 요청을 우선합니다.
필요할 때만 이 문서를 참고하고, 새로운 판단이 필요하면 현재 상황을 다시 확인합니다.

## Codex 프로젝트 표준 폴더

Codex 작업용 프로젝트는 바탕화면이 아니라 사용자 문서 폴더 아래에서 관리합니다.

```text
~/Documents/Codex_프로젝트
├── 01_패밀리스케줄러
├── 02_광양창의융합
├── 03_포스코_신입사원_금융교육
├── 04_제주연수원_예약확인
├── 05_Mac_정리_설정
└── 06_메일발송
```

집이나 다른 PC에서 다시 시작할 때는 기존 Codex 작업 폴더를 삭제하기 전에 GitHub에 없는 로컬 변경사항이 있는지 먼저 확인하고, GitHub 백업 프로젝트는 위 구조로 다시 받습니다.
`06_메일발송`처럼 `.env`가 들어 있는 로컬 전용 작업 폴더는 GitHub에 올리기 전에 민감정보 제거 여부를 먼저 확인합니다.

## 문서 분류 기준

문서 관리는 ~/Documents를 중심으로 합니다.

- 00_받은문서: 아직 분류하지 않은 임시 문서
- 01_염경섭: 염경섭 개인 문서, 사진, 관련 자료
- 02_염초롱: 염초롱 개인 문서, 학교/교육 관련 문서, 관련 자료
- 03_포스코새마을금고: 금고, 금융교육, 신입사원 교육, 업무 자료
- 04_디자인: PPT, 피그마, 이미지 소스, 폰트, 디자인 자료, 스크린샷
- 99_기타: 위 기준에 애매한 문서

99_기타에 같은 종류가 반복해서 쌓이면 새 분류 폴더로 독립시킵니다.

## 파일명 기준

일반 문서 파일은 폴더 분류를 우선하고, 파일명에는 날짜와 분류명을 기본으로 붙이지 않습니다.
날짜는 Finder의 수정일/생성일로 확인합니다.

날짜나 버전이 꼭 필요한 계약서, 영수증, 납부확인서, 반복되는 신청서, 디자인 시안만 파일명 뒤에 짧게 붙입니다.

예시:

- 주민등록등본.pdf
- 신청서.pdf
- 예금안내.pdf
- 명함시안_v1.pdf
- 영수증_2026-05.pdf

프로젝트 내부 파일, 이미지 소스, 폰트, 코드 파일은 연결이 깨질 수 있으므로 필요할 때만 별도로 정리합니다.

## Windows PC에서 쓰는 방법

다른 Windows PC에서도 같은 폴더 구조를 만들 수 있도록 `windows/setup-folders.ps1`을 제공합니다.

Windows에서 이 저장소를 내려받은 뒤 PowerShell에서 실행합니다.

```powershell
powershell -ExecutionPolicy Bypass -File .\windows\setup-folders.ps1
```

이 작업은 없는 폴더만 만들고, 기존 파일이나 폴더는 삭제하지 않습니다.
